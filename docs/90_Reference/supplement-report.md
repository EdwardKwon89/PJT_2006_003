# SUPPLEMENT 1. Board 의사결정 게이트 체계

## 1.1 의사결정 게이트 설계 원칙

프로젝트 전 과정에서 Board(나)의 개입이 필요한 시점을 사전에 정의하고, 각 게이트에서 확인해야 할 산출물과 승인 기준을 명확히 한다. Agent가 자율적으로 작업을 수행하되, 핵심 방향 결정은 반드시 Board가 통제한다.

```
[게이트 유형 3가지]

G-HARD (필수 정지 게이트)
  → Board 승인 없이 다음 단계 진행 불가
  → CEO Agent가 Approval 요청 티켓 생성
  → Board가 명시적으로 "승인" 코멘트 필요

G-REVIEW (검토 게이트)
  → Board가 산출물 검토 후 피드백 제공
  → 3영업일 내 피드백 없으면 자동 진행
  → 중요 이슈 발견 시 G-HARD로 격상 가능

G-INFO (정보 공유 게이트)
  → Board에게 현황 보고 (승인 불필요)
  → 주간 보고서 포함
  → 이슈 없으면 자동 진행
```

산출물:
  REQUIREMENTS.md (GSD 기반)

  MVP Scope 정의서

           ↓
    기능·프로세스 기준선 정의
